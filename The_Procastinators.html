            Risks

            Native vs Web vs Hybrid

            While developing our application there were a lot of questions, we had to ask ourselves as a team, the biggest question being if we want our application to be native, hybrid or a web application. Initially, as we looked back at our assignment 2, we took a more native approach when discussing how to develop the application and the tools and technologies we would need. Upon further research we realised it was not the best decision. If we look at some of the best applications found in the Apple store/play store, we see that most of them are native applications. Native applications are known to have the best user experience in terms of graphics, user reaction time and have some of the best app performance. We were hind sighted by these aspects and ignored the fact that they are really costly to develop and require a very steep maintenance cost. Furthermore, we require a different code basis for each platform we choose which is very time consuming. After evaluating all our choices, we decided to make our application hybrid. It seems the most practical to do with our resources and level of skills now. To build a hybrid application we would require knowing JavaScript, HTML and CSS and only one code base is required on all platforms (IOS, android). Additionally, we get to retain the same ability to access device features like native applications. However, Hybrid applications have a disadvantage when it comes to performance in terms of graphics, usability, and reaction time which we must keep in mind. We also realise that cross platform development is tough and getting our app to work seamlessly on each would require a lot of work.

            Functional elements

            Interface: The design of hybrid applications is different from that of native apps. As a result, the user interface is less smooth. Possibilities are also limited owing to the usage of WebView, which prevents the full potential of the devices from being realised. In order to access a device's hardware features (microphone, camera, contacts, etc.) for which the native apps are installed, it is possible to include native elements of each platform’s user interfaces (iOS, Android): it is inevitable that native code will be used to access the specific features in order to create a seamless user experience. But this in turn becomes a hassle when it comes to the testing stage. Furthermore, there is a huge dependency on the browser's speed.

            Security concerns: Many developers are concerned about hybrid mobile app security since these apps take more effort to safeguard. Because JavaScript and HTML are easier to reverse engineer and change, hybrid apps may be more vulnerable to cyberattacks than native binary code programmes. Through man-in-the-middle attacks, hackers can get access to a hybrid app's database and customer information.
